# Workflow Plan

Use this checklist to track implementation progress.

## Phase 1: Content Foundation

- [ ] Collect final experience entries (roles, dates, impact bullets, technologies).
- [ ] Collect final certifications (issuer, date, credential links, scope).
- [ ] Collect final project entries (problem, solution, architecture, outcomes, links).

## Phase 2: Route Completion

- [ ] Add experience slug pages and connect listing to details.
- [ ] Add certifications slug pages and connect listing to details.
- [ ] Add projects slug pages and connect listing to details.

## Phase 3: Documentation Engine

- [ ] Add MDX content directory structure for documentation topics.
- [ ] Implement MDX loading and rendering pipeline.
- [ ] Connect `/documentation/[...slug]` to real MDX content.

## Phase 4: Content Polish

- [ ] Replace placeholder section copy with final website copy.
- [ ] Add real external profile/repo/certification links.
- [ ] Review metadata and route titles/descriptions for SEO quality.

## Phase 5: Final Validation

- [ ] Run local dev checks (`npm run dev`) and verify all routes.
- [ ] Run production build (`npm run build`) and resolve any issues.
- [ ] Review visual consistency and mobile responsiveness.
